# PART 2 — Authentication & Authorization Security Fix

## Current Issue

Currently:

* A normal user logs in successfully.
* When the user manually navigates to `/admin` or any admin route, backend correctly denies access.
* However, authentication cookies remain valid.
* The application redirects the user back to the dashboard.
* The user remains logged in.

This is a security issue.

Implement a production-ready authentication flow.

---

## Backend Middleware

Create or update middleware:

* `requireAuth`
* `requireAdmin`

Every admin route must be protected.

Example:

```ts
router.use(requireAuth);
router.use(requireAdmin);
```

or

```ts
router.get("/users", requireAuth, requireAdmin, controller);
```

The middleware must verify:

```ts
user.role === "admin"
```

---

## Unauthorized Admin Access

If a logged-in user is NOT an admin:

Immediately:

* Invalidate JWT
* Destroy session
* Clear HTTP-only cookies
* Remove refresh token (if used)
* Clear authentication data
* Return HTTP 401

Example:

```ts
return res.status(401).json({
    success: false,
    message: "Unauthorized",
    redirect: "/signin"
});
```

Never redirect to the dashboard.

Never preserve authentication.

Never keep cookies alive.

---

## Logout Endpoint

Update logout endpoint to:

* Clear access token cookie
* Clear refresh token cookie
* Destroy session
* Remove refresh token from database (if applicable)
* Return success response

Use production cookie settings:

```ts
httpOnly: true,
sameSite: "lax",
secure: process.env.NODE_ENV === "production"
```

# PART 3 — React Authentication Flow

Update the Auth Context.

Whenever any request returns:

```ts
401 Unauthorized
```

Immediately:

```ts
localStorage.clear();
sessionStorage.clear();
queryClient.clear();
resetAuthState();
logout();
window.location.replace("/signin");
```

Remove all cached user information.

Never redirect to the dashboard.

Refreshing the browser must never restore the previous session.

---

## Axios Interceptor

Create or update a global Axios response interceptor:

```ts
axios.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) {
      localStorage.clear();
      sessionStorage.clear();
      window.location.replace("/signin");
    }
    return Promise.reject(error);
  }
);
```

This should work globally for the entire application.

---

## Route Guards

Protect every admin route:

* `/admin`
* `/admin/dashboard`
* `/admin/users`
* `/admin/jobs`
* `/admin/settings`
* Any future `/admin/*` route

Non-admin users must never access admin pages.

Unauthorized users must always be redirected only to:

```
/signin
```

Never redirect to the dashboard.

Prevent session restoration after browser refresh.
